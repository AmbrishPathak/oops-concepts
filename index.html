Object-Oriented Programming (OOP) in Python

Core concepts and syntax for your next technical interview.

The Four Pillars of OOP

Abstraction:

Hiding complex implementation details and showing only the essential features of the object.

Encapsulation:

Bundling data (attributes) and methods that operate on the data into a single unit (class). Restricts direct access to components.

Inheritance:

Mechanism for creating a new class (child) from an existing class (parent), inheriting its attributes and methods.

Polymorphism:

The ability of different classes to be treated as instances of the same class through a common interface.

Class and Object Syntax

class Dog:
    # Class variable (shared among all instances)
    species = "Canis familiaris"

    # The constructor method
    def __init__(self, name, age):
        # Instance variables (unique to each instance)
        self.name = name
        self.age = age

# Creating an object (instance)
my_dog = Dog("Rex", 5)
print(my_dog.name)        # Rex
print(my_dog.species)   # Canis familiaris

Common Dunder Methods

class Book:
    def __init__(self, title, author):
        self.title, self.author = title, author
    
    # For user-facing output (e.g., print())
    def __str__(self):
        return f"{self.title} by {self.author}"

    # For developer-facing output (debugging)
    def __repr__(self):
        return f"Book('{self.title}')"

    # For equality checking
    def __eq__(self, other):
        return self.title == other.title

Class & Static Methods

class Calculator:
    @staticmethod
    # Doesn't access class or instance state
    def add(x, y):
        return x + y

    @classmethod
    # Accesses class state (cls), not instance state
    def create_doubler(cls, value):
        # A factory method
        return cls.add(value, value)

Calculator.add(5, 3) # 8
Calculator.create_doubler(10) # 20

Inheritance: Reusing Code

class Animal:
    def __init__(self, name):
        self.name = name
    def speak(self): return "Sound"

class Cat(Animal):
    def __init__(self, name, breed):
        super().__init__(name) # Call parent constructor
        self.breed = breed
    
    def speak(self): # Method overriding
        return "Meow"

Encapsulation: Protecting Data

class BankAccount:
    def __init__(self, initial_deposit):
        # Private variable with name mangling
        self.__balance = initial_deposit

    # Getter using property decorator
    @property
    def balance(self):
        return self.__balance

    # Setter for the property
    @balance.setter
    def balance(self, amount):
        if amount >= 0: self.__balance = amount

Polymorphism & Duck Typing

class Dog:
    def speak(self): return "Woof!"

class Duck:
    def speak(self): return "Quack!"

# This function works with any object that has a .speak() method
def make_it_speak(animal):
    print(animal.speak())

make_it_speak(Dog())   # Woof!
make_it_speak(Duck()) # Quack!

Abstraction: Defining a Blueprint

from abc import ABC, abstractmethod

class Shape(ABC): # Abstract Base Class
    @abstractmethod
    def area(self): pass

class Square(Shape):
    def __init__(self, side): self.side = side
    def area(self): return self.side ** 2

# s = Shape() # Raises TypeError
sq = Square(5); sq.area() # 25

Common Interview Questions

Class vs. Instance Variables?

Class variables are shared by all instances. Instance variables are unique to each instance, defined in `__init__`.

What is Method Resolution Order (MRO)?

In multiple inheritance, MRO is the order Python searches for a method in a class's hierarchy. View with `ClassName.mro()`.

Difference between `__str__` and `__repr__`?

`__str__` is for creating a readable, user-friendly output. `__repr__` is for creating an unambiguous, developer-friendly representation of the object, which should ideally be valid Python code to recreate the object.
